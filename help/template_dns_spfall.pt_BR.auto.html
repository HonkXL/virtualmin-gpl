O registro SPF cobre todos os remetentes?
Este campo controla como o registro SPF cobre outros hosts além do próprio servidor Virtualmin e hosts especificados nas opções anteriores. As opções são:
Não (modo SPF ? todos )
O registro SPF não cobre todos os possíveis remetentes. Isto deverá ser selecionado se os usuários precisarem enviar de servidores de correio arbitrários (como os de seu ISP). No entanto, é provável que isso leve a efeitos colaterais infelizes no futuro, à medida que os servidores de e-mail começarem a bloquear hosts sem registro SPF ou com registros SPF particularmente frouxos.

Sim (modo SPF ~todos )
O registro SPF será aplicado apenas ao próprio servidor Virtualmin e a outros hosts definidos acima. Este é o método preferido de uso do SPF, pois limita os hosts que podem enviar mensagens alegando ser deste domínio, mas não bloqueia totalmente mensagens de outras fontes.

Sim, e negar outros remetentes (modo SPF -all )
Como a opção acima, mas os repientes devem descartar totalmente as mensagens de outros remetentes.

Leia mais sobre o SPF na página inicial do Sender Policy Framework .

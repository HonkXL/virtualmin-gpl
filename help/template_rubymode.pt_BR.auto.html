Modo de execução Ruby padrão
Esta opção determina se os scripts Ruby em novos servidores virtuais são executados como o usuário do servidor web Apache (normalmente httpd ou apache ) ou o usuário Unix que possui o servidor virtual. O primeiro é mais rápido, pois significa que nenhum processo adicional precisa ser iniciado para executar o código Ruby, mas é menos seguro, pois os scripts leem e gravam arquivos como um usuário diferente do proprietário do domínio.

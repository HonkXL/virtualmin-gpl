Criar domínio padrão de host com certificado Let's Encrypt?
Quando esta opção estiver definida como Sim , o Virtualmin tentará automaticamente criar um domínio padrão de host oculto (servidor virtual) com um certificado Let's Encrypt durante a próxima verificação de configuração. Se o certificado Let's Encrypt não puder ser solicitado com êxito para o nome do host do sistema, o domínio padrão do host será removido automaticamente.

No entanto, é possível usar a opção Sim e manter visível para manter o domínio padrão do host visível e não removê-lo caso o certificado Let's Encrypt não possa ser solicitado com sucesso, o que permitirá ajustes manuais posteriormente.

Se o nome do host do sistema mudar, o Virtualmin tentará automaticamente atualizar o nome de domínio padrão do host de acordo na próxima verificação de configuração.

O domínio padrão do host pode ser desabilitado configurando esta opção como Não, o que fará com que ele seja removido do sistema na próxima verificação de configuração.

Esteja ciente de que o uso do domínio padrão do host para correio é altamente desencorajado e pode causar problemas na entrega de correio!

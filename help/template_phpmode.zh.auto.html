默认 PHP 执行模式
默认 PHP 脚本执行模式定义了 PHP 代码在新虚拟服务器中的处理方式。

禁用模式意味着服务器不执行 PHP 脚本，而是将其视为纯文本文件。当不需要 PHP 处理或将 PHP 代码作为可下载文件提供时，可以使用此模式。

FPM是首选模式，可提供高效率和自适应流程管理等高级功能。它作为独立的 FastCGI 服务器运行，由于其强大的性能，非常适合高流量站点。

FCGId对于 FPM 不可用的服务器很有用。它还支持每个虚拟主机的配置，尽管它在可扩展性和管理功能方面通常无法与 FPM 相匹配。

CGI Wrapper模式通过 CGI 协议将每个 PHP 脚本作为单独的进程运行。由于为每个请求创建新进程的开销，此模式的效率低于 FPM 或 FCGId。由于其性能限制，它很少在现代设置中使用。

mod_php将 PHP 直接集成到 Apache Web 服务器中，允许 PHP 脚本在服务器自己的进程中执行。然而，由于安全和隔离问题，加上支持和更新的停止，它现在已经过时了。

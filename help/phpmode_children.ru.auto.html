Максимальное количество подпроцессов службы PHP
Этот параметр устанавливает ограничение на количество одновременных запросов, которые будут обслуживаться, когда PHP-скрипты для этого домена настраиваются для запуска через FPM, количество дочерних процессов будет создано, когда для параметра pm установлено значение static и максимальное количество дочерних процессов потенциально может быть создан, если для параметра pm установлено значение dynamic . Максимальное количество подпроцессов не может быть меньше 5, если значение параметра pm.max_spare_servers по умолчанию также не уменьшено вручную. Настоятельно не рекомендуется устанавливать для этого параметра значение Unlimited при работе в режиме FPM.

Когда скрипты PHP для этого домена настроены на запуск через FCGId, количество процессов PHP, заданное в этом поле, будет постоянно работать для обслуживания запросов. Вы можете увеличить это значение по умолчанию, равное 20, чтобы уменьшить задержку PHP-скрипта, или уменьшить его, чтобы уменьшить использование памяти. Установка значения Unlimited в режиме FCGId приведет к тому, что процессы PHP будут запускаться только по мере необходимости по запросу и очищаться после некоторого периода бездействия.

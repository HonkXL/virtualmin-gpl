Максимальное количество подпроцессов службы PHP
Этот параметр устанавливает ограничение на количество одновременных запросов, которые будут обслуживаться, когда PHP-скрипты для этого домена настраиваются для запуска через FPM, количество дочерних процессов будет создано, когда для параметра pm установлено значение static и максимальное количество дочерних процессов потенциально может быть создан, если для параметра pm установлено значение dynamic . Максимальное количество подпроцессов не может быть меньше pm.max_spare_servers , которое по умолчанию равно 5.

Когда PHP-скрипты для этого домена настраиваются для запуска через FCGId, управление дочерними процессами PHP всегда должно быть отключено, что будет направлять только один запрос за раз к порожденным им процессам приложения. По умолчанию и с настройкой переменной среды PHP_FCGI_CHILDREN=0 управление дочерними процессами PHP отключено.

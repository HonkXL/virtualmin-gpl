デフォルトのPHP実行モード
デフォルトの PHP スクリプト実行モードは、新しい仮想サーバーで PHP コードがどのように処理されるかを定義します。

無効モードは、PHP スクリプトがサーバーによって実行されず、プレーン テキスト ファイルとして扱われることを意味します。このモードは、PHP 処理が必要ない場合、または PHP コードをダウンロード可能なファイルとして提供する場合に使用できます。

FPMが推奨モードであり、高効率と適応プロセス管理などの高度な機能を提供します。スタンドアロンの FastCGI サーバーとして動作し、その堅牢なパフォーマンス機能によりトラフィックの多いサイトに最適です。

FCGId は、 FPM が利用できないサーバーに役立ちます。また、仮想ホストごとの構成もサポートしていますが、一般的にスケーラビリティと管理機能の点で FPM に匹敵しません。

CGI ラッパーモードは、CGI プロトコルを通じて各 PHP スクリプトを別個のプロセスとして実行します。このモードは、リクエストごとに新しいプロセスを作成するオーバーヘッドのため、FPM や FCGId よりも効率が低くなります。パフォーマンス上の制限があるため、最新のセットアップではほとんど使用されません。

mod_php はPHP を Apache Web サーバーに直接統合し、サーバー独自のプロセス内で PHP スクリプトを実行できるようにします。ただし、セキュリティと分離の問題、さらにサポートとアップデートの中止により、現在は廃止されています。

Sous-processus maximum du service PHP
Cette option définit la limite du nombre de requêtes simultanées qui seront servies, lorsque les scripts PHP pour ce domaine sont configurés pour s'exécuter via FPM, le nombre de processus enfants sera créé lorsque l'option pm est définie sur static et le nombre maximum de processus enfants pourrait potentiellement être créé lorsque l'option pm est définie sur dynamic . Le nombre de sous-processus maximum ne peut pas être inférieur à pm.max_spare_servers qui est par défaut à 5.

Lorsque les scripts PHP pour ce domaine sont configurés pour s'exécuter via FCGId, la gestion des processus enfants PHP doit toujours être désactivée, ce qui n'achemine qu'une requête à la fois vers les processus d'application qu'elle a générés. Par défaut, et avec le paramètre de variable d'environnement PHP_FCGI_CHILDREN=0 , la gestion des processus enfants PHP est désactivée.

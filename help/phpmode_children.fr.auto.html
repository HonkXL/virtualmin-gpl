Sous-processus maximum du service PHP
Cette option définit la limite du nombre de requêtes simultanées qui seront servies, lorsque les scripts PHP pour ce domaine sont configurés pour s'exécuter via FPM, le nombre de processus enfants sera créé lorsque l'option pm est définie sur static et le nombre maximum de processus enfants pourrait potentiellement être créé lorsque l'option pm est définie sur dynamic . Le nombre maximal de sous-processus ne peut pas être inférieur à 5, sauf si la valeur par défaut de l'option pm.max_spare_servers est également diminuée manuellement. Il est fortement déconseillé de définir cette option sur Illimité lors d'une exécution en mode FPM.

Lorsque les scripts PHP pour ce domaine sont configurés pour s'exécuter via FCGId, le nombre de processus PHP défini dans ce champ sera exécuté en permanence pour répondre aux demandes. Vous pouvez l'augmenter à partir de la valeur par défaut de 20 pour améliorer la latence des scripts PHP, ou la diminuer pour réduire l'utilisation de la mémoire. Si vous le définissez sur Illimité en mode FCGId, les processus PHP ne seront lancés qu'en cas de besoin et à la demande, et seront nettoyés après une certaine période d'inactivité.

Maximale Unterprozesse des PHP-Dienstes
Diese Option legt die Grenze für die Anzahl gleichzeitiger Anforderungen fest, die bedient werden, wenn PHP-Skripts für diese Domäne so eingerichtet sind, dass sie über FPM ausgeführt werden, die Anzahl der untergeordneten Prozesse wird erstellt, wenn die pm -Option auf static eingestellt ist, und die maximale Anzahl untergeordneter Prozesse könnte möglicherweise erstellt werden, wenn die Option pm auf dynamic gesetzt ist. Die Anzahl der maximalen Unterprozesse darf nicht niedriger sein als pm.max_spare_servers , die standardmäßig auf 5 eingestellt ist.

Wenn PHP-Skripte für diese Domain über FCGId ausgeführt werden sollen, sollte die untergeordnete PHP-Prozessverwaltung immer deaktiviert werden, wodurch jeweils nur eine Anfrage an die von ihr erzeugten Anwendungsprozesse weitergeleitet wird. Standardmäßig und mit der Einstellung der Umgebungsvariablen PHP_FCGI_CHILDREN=0 ist die PHP-Verwaltung untergeordneter Prozesse deaktiviert.

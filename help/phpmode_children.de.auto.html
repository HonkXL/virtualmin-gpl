Maximale Unterprozesse des PHP-Dienstes
Diese Option legt die Grenze für die Anzahl gleichzeitiger Anforderungen fest, die bedient werden, wenn PHP-Skripts für diese Domäne so eingerichtet sind, dass sie über FPM ausgeführt werden, die Anzahl der untergeordneten Prozesse wird erstellt, wenn die pm -Option auf static eingestellt ist, und die maximale Anzahl untergeordneter Prozesse könnte möglicherweise erstellt werden, wenn die Option pm auf dynamic gesetzt ist. Die Anzahl der maximalen Unterprozesse kann nicht kleiner als 5 sein, es sei denn, der Standardwert pm.max_spare_servers Option wird ebenfalls manuell verringert. Es wird dringend davon abgeraten, diese Option auf Unbegrenzt zu setzen, wenn sie im FPM-Modus ausgeführt wird.

Wenn PHP-Skripte für diese Domain so eingerichtet sind, dass sie über FCGId ausgeführt werden, wird die in diesem Feld festgelegte Anzahl von PHP-Prozessen ständig ausgeführt, um Anfragen zu bedienen. Sie können die Standardeinstellung von 20 erhöhen, um die PHP-Skriptlatenz zu verbessern, oder sie verringern, um die Speichernutzung zu reduzieren. Wenn Sie es im FCGId-Modus auf Unlimited setzen, werden PHP-Prozesse nur bei Bedarf gestartet und nach einer gewissen Zeit der Inaktivität bereinigt.

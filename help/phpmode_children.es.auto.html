Subprocesos máximos del servicio PHP
Esta opción establece el límite en la cantidad de solicitudes simultáneas que se atenderán, cuando los scripts PHP para la configuración de este dominio se ejecuten a través de FPM, la cantidad de procesos secundarios se creará cuando la opción pm se establezca en static y la cantidad máxima de procesos secundarios podría crearse potencialmente cuando la opción pm se establece en dynamic . El número máximo de subprocesos no puede ser inferior a 5, a menos que el valor predeterminado de la opción pm.max_spare_servers también se reduzca manualmente. No se recomienda establecer esta opción en Ilimitado cuando se ejecuta en modo FPM.

Cuando los scripts PHP para la configuración de este dominio se ejecuten a través de FCGId, la cantidad de procesos PHP establecidos en este campo se mantendrán en ejecución en todo momento para atender las solicitudes. Puede aumentar esto desde el valor predeterminado de 20 para mejorar la latencia del script PHP o disminuirlo para reducir el uso de memoria. Configurarlo en Ilimitado en el modo FCGId hará que los procesos de PHP se inicien solo cuando sea necesario y se limpien después de un período de inactividad.

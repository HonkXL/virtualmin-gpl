Subprocesos máximos del servicio PHP
Esta opción establece el límite en la cantidad de solicitudes simultáneas que se atenderán, cuando los scripts PHP para la configuración de este dominio se ejecuten a través de FPM, la cantidad de procesos secundarios se creará cuando la opción pm se establezca en static y la cantidad máxima de procesos secundarios podría crearse potencialmente cuando la opción pm se establece en dynamic . El número máximo de subprocesos no puede ser inferior a pm.max_spare_servers cuyo valor predeterminado es 5.

Cuando los scripts de PHP para la configuración de este dominio se ejecuten a través de FCGId, la gestión de procesos secundarios de PHP siempre debe estar deshabilitada, lo que solo enrutará una solicitud a la vez a los procesos de aplicación que ha generado. De forma predeterminada, y con la configuración de la variable de entorno PHP_FCGI_CHILDREN=0 , la gestión de procesos secundarios de PHP está deshabilitada.

PHP服務最大子進程數
此選項設置將服務的同時請求數的限制，當此域的 PHP 腳本設置為通過 FPM 運行時，當pm選項設置為static時將創建子進程數和最大子進程數當pm選項設置為dynamic時可能會創建。最大子進程數不能低於 5，除非也手動減少默認pm.max_spare_servers選項值。在 FPM 模式下運行時，強烈建議不要將此選項設置為Unlimited 。

當此域的 PHP 腳本設置為通過 FCGId 運行時，此字段中設置的 PHP 進程數將始終保持運行以服務請求。您可以從默認值 20 增加此值以改善 PHP 腳本延遲，或減少它以減少內存使用。在 FCGId 模式下將其設置為Unlimited將導致 PHP 進程僅根據需要啟動，並在一段時間不活動後被清理。

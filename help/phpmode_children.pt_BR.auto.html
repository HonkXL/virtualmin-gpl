Subprocessos máximos do serviço PHP
Esta opção define o limite do número de solicitações simultâneas que serão atendidas, quando os scripts PHP para este domínio forem configurados para execução via FPM, o número de processos filho será criado quando a opção pm estiver definida como static e o número máximo de processos filho poderia ser criado quando a opção pm é definida como dynamic . O número máximo de subprocessos não pode ser inferior a pm.max_spare_servers cujo padrão é 5.

Quando os scripts PHP para este domínio são configurados para serem executados via FCGId, o gerenciamento de processos filho do PHP deve sempre ser desabilitado, o que direcionará apenas uma solicitação por vez para os processos do aplicativo que ele gerou. Por padrão, e com a configuração da variável de ambiente PHP_FCGI_CHILDREN=0 , o gerenciamento do processo filho do PHP está desabilitado.

Subprocessos máximos do serviço PHP
Esta opção define o limite no número de solicitações simultâneas que serão atendidas, quando os scripts PHP para este domínio forem configurados para execução via FPM, o número de processos filho será criado quando a opção pm estiver definida como static e o número máximo de processos filho poderia ser criado quando a opção pm é definida como dynamic . O número máximo de subprocessos não pode ser inferior a 5, a menos que o valor da opção pm.max_spare_servers padrão também seja reduzido manualmente. Definir esta opção como Ilimitado não é altamente recomendado quando executado no modo FPM.

Quando os scripts PHP para este domínio forem configurados para execução via FCGId, o número de processos PHP configurados neste campo será mantido em execução o tempo todo para atender às solicitações. Você pode aumentar isso do padrão de 20 para melhorar a latência do script PHP ou reduzi-lo para reduzir o uso de memória. Configurá-lo como Ilimitado no modo FCGId fará com que os processos PHP sejam iniciados apenas quando necessário sob demanda e sejam limpos após algum período de inatividade.
